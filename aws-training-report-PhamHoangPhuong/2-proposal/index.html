Proposal

Project: SaaS HR Multi-Tenant Microservices

AWS-based Multi-Tenant SaaS Human Resource Management System

1. Executive Summary

SaaS HR Multi-Tenant is a comprehensive human resource management platform designed for multiple organizations (multi-tenant), built on a modern microservices architecture. The system supports employee management, attendance tracking, leave management, system health monitoring, and tenant administration in a secure, scalable, and cost-effective manner.

Core Technologies:

  • Frontend: ReactJS (Vite) – Modern Single Page Application.
  • Backend: Python FastAPI microservices (auth-service, tenant-service, hr-service).
  • API Gateway: Nginx → Application Load Balancer (ALB).
  • Database: Amazon RDS MySQL Multi-AZ using the Pool Model (logical tenant separation).
  • Authentication: AWS Cognito + JWT RS256 (asymmetric authentication).
  • Asynchronous Communication: Amazon SQS.
  • Deployment: AWS 3-tier architecture (CloudFront + S3 + ECS Fargate + RDS) in the ap-southeast-1 Region.

The platform is designed for rapid deployment using the AWS Management Console, allowing easy scalability while ensuring strict tenant data isolation, high security, and reasonable operating costs (~USD 130/month for a 24/7 production environment). The project is suitable for startups, laboratories, or enterprises that require an internal HR SaaS platform with future commercialization potential.

2. Problem Statement

Current Challenges

Many organizations still rely on traditional HR systems or standalone tools such as Microsoft Excel, Google Sheets, or on-premises software, resulting in:

  • Difficulty managing multiple organizations (multi-tenant).
  • Lack of secure authentication and data isolation.
  • High infrastructure integration costs.
  • Manual HR processes such as attendance and leave management.
  • Limited scalability and monitoring as the number of tenants increases.

Proposed Solution

Develop a SaaS HR Multi-Tenant platform on AWS using a microservices architecture, a pooled database model, AWS Cognito with asymmetric authentication, and a modern management interface. Each tenant owns an independent subdomain while securely sharing the same infrastructure.

Benefits

  • Accelerates HR management for multiple organizations.
  • Reduces operating costs through ECS Fargate container architecture.
  • Provides high availability with Multi-AZ deployment.
  • Supports future feature expansion such as AI attendance and automated reporting.
  • Can be reused for commercial or research purposes.

Return on Investment (ROI)

The platform significantly reduces manual administrative effort and infrastructure costs compared to on-premises deployment. Estimated production operating cost is approximately USD 130 per month, with an expected return on investment within 6–12 months through improved operational efficiency and SaaS commercialization.

3. Solution Architecture

Architecture Overview

  • Edge Layer: CloudFront + Amazon S3 + AWS Certificate Manager (ACM).
  • Application Layer: Application Load Balancer + Amazon ECS Fargate (3 microservices).
  • Data Layer: Amazon RDS MySQL Multi-AZ.

Main Components

  • Frontend: ReactJS (Vite) static website hosted on Amazon S3 and distributed through CloudFront.
  • API Gateway: ALB routes requests to /api/v1/auth/*, /api/v1/tenants/*, and /api/v1/hr/*.
  • Microservices:
    • auth-service: Authentication, JWT RS256, AWS Cognito integration.
    • tenant-service: Tenant, subscription, and subdomain management.
    • hr-service: Employee, department, attendance, and leave management using the Pool Model.
  • Database: Three logical databases (auth_db, tenant_db, hr_db) on a single Amazon RDS Multi-AZ instance.
  • Asynchronous Communication: Amazon SQS (tenant → hr).
  • Identity & Secrets: AWS Cognito and AWS Systems Manager Parameter Store.
  • Monitoring: Amazon CloudWatch with Amazon SNS alerts.
  • Networking: One VPC, two Availability Zones, six subnets, and one cost-optimized NAT Gateway.

Architecture Design

  • Tenant isolation using custom:tenant_id in JWT and logical data separation.
  • High availability through Multi-AZ deployment for ECS and RDS.
  • Security with Zero Trust Security Groups, private subnets, and Origin Access Control (OAC) for Amazon S3.

4. Technical Implementation

Implementation Phases

  1. Preparation & Design (Month 0–1): Finalize architecture, estimate costs, and prepare source code.
  2. Infrastructure Deployment (Month 1): Create VPC, subnets, security groups, RDS, Cognito, SQS, and ECR using the AWS Management Console.
  3. Microservices Deployment (Month 2): Build and push Docker images, create ECS task definitions and services, then configure ALB and CloudFront.
  4. Integration & Testing (Month 2): Replace Redis with Amazon SQS, perform end-to-end testing, and configure monitoring.
  5. Optimization & Operations (Month 3): Apply FinOps practices, configure alarms, and prepare documentation.

Technical Requirements

  • AWS services: VPC, ECS Fargate, RDS, Cognito, CloudFront.
  • FastAPI, ReactJS, Docker, and AWS CLI.
  • Knowledge of JWT, multi-tenancy, and AWS security best practices.

5. Project Roadmap

  • Month 0 (Preparation): Design the architecture, migrate from Docker Compose to AWS, and estimate costs.
  • Internship (Months 1–3):
    • Month 1: Deploy networking and database infrastructure.
    • Month 2: Deploy microservices, frontend, authentication, and asynchronous communication.
    • Month 3: Perform testing, optimize costs, and deliver the project.
    • Post-deployment: Monitor operations, extend features, and research AI and tenant billing.

6. Budget Estimation

Infrastructure Costs

  • NAT Gateway: USD 43–50/month
  • Amazon RDS MySQL Multi-AZ: USD 30–38/month
  • Application Load Balancer: USD 22–26/month
  • Amazon ECS Fargate: USD 12–18/month
  • Data Transfer + CloudWatch: USD 5–12/month
  • CloudFront + Amazon S3 + Amazon ECR + Others: USD 3–8/month

Total: Approximately USD 115–155 (~USD 130/month)

FinOps Notes

  • The highest costs come from NAT Gateway and Amazon RDS Multi-AZ.
  • The infrastructure can be destroyed after demonstrations to reduce costs to only a few dollars.
  • AWS Budgets is configured to send alerts when spending exceeds 80%.

7. Risk Assessment

Risk Matrix

  • Availability Zone failure: Medium to High impact. Mitigated through Multi-AZ deployment.
  • Budget overrun: Medium impact. Mitigated using AWS Budgets and resource cleanup.
  • Security breaches or data leakage: High impact. Mitigated using Cognito, Zero Trust Security Groups, and private subnets.
  • Performance degradation as tenants grow: Medium impact. Mitigated through CloudWatch monitoring and stateless architecture.

Contingency Plan

  • Rollback using CloudFormation exports or rebuild infrastructure manually.
  • Enable automatic Amazon RDS backups for seven days.
  • Run the application locally with Docker Compose if AWS becomes unavailable.

8. Expected Outcomes

  • Build a modern, scalable, and commercial-ready multi-tenant HR SaaS platform.
  • Create a reusable platform for internal enterprise use or SaaS commercialization while supporting cloud and microservices research.
  • Gain practical experience with AWS 3-tier architecture, Amazon ECS Fargate, multi-tenancy, and FinOps.
  • Establish a strong foundation for future SaaS projects.
