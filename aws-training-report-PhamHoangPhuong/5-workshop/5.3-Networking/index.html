Networking

Building the Network Foundation First: One VPC, 6 Subnets across 2 AZs, one Internet Gateway, one NAT Gateway, and Security Groups.

Step 1 — VPC + 6 Subnets + IGW + NAT Gateway:

Navigate to VPC → click Create VPC — Name: saashr-vpc, CIDR Block: 10.0.0.0/16.

  • VPC -> Successfully created VPC dashboard view

  • Subnets — Provisioning 6 distinct subnets:

  • Navigate to VPC -> Subnets -> click Create Subnet

  • 6 subnets successfully provisioned

  • Internet Gateway — Create saashr-igw, then attach it to your VPC. VPC -> Internet gateways -> click Create internet gateway

    Click Actions -> Attach to VPC
  • NAT Gateway — Create 1 NAT Gateway inside public-1a subnet and allocate an Elastic IP. Go to VPC -> Elastic IP addresses -> click Allocate Elastic IP address

  • Click Allocate Next, go to VPC -> NAT gateways -> click Create NAT gateway

    NAT Gateway creation complete

  • Route Tables Configuration:

    rt-public → Target: 0.0.0.0/0 → Associated with: Internet Gateway (IGW) → Explicitly associate with public-1a, public-1b subnets.

    rt-app → Target: 0.0.0.0/0 → Associated with: NAT Gateway → Explicitly associate with app-1a, app-1b subnets.

    rt-data → No internet routing targets (Local only) → Explicitly associate with data-1a, data-1b subnets.

    Go to VPC -> Route tables -> click Create route table

  • Click Actions -> Edit subnet associations

  • Click Edit routes

  • 3 Route tables successfully configured

Step 2 — Security Groups Configuration (Zero-Trust Model)

Provisioning 3 core security groups:

  • ALB Security Group: Inbound and Outbound rules

  • ECS Security Group: Inbound and Outbound rules

  • RDS Security Group: Inbound and Outbound rules

    The security groups are structurally chained together (alb → ecs → rds). This design guarantees that the database layer remains completely inaccessible from the public internet—allowing ingress traffic strictly originating from the active ECS tasks.
