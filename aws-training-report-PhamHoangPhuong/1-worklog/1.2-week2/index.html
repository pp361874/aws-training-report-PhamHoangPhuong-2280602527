Week 2 Worklog

Week 2 Objectives:

  • Master the use of AWS CLI to manage AWS resources through the command-line interface, enabling automation of administrative tasks.
  • Learn and deploy AWS database services, including Amazon RDS (relational database) and Amazon DynamoDB (NoSQL database).
  • Understand the differences between relational and non-relational database models to select the appropriate service for different application requirements.
  • Become familiar with Amazon Lightsail for rapidly deploying common web applications and understand the benefits of this simplified PaaS service.
  • Study infrastructure auto scaling using Amazon EC2 Auto Scaling and Elastic Load Balancing (ELB) to improve system availability and scalability.
  • Continue developing AWS deployment skills through hands-on labs and FCAJ workshops.

Week 2 Achievements:

  • Expanded my knowledge of AWS resource management through AWS CLI and gained a deeper understanding of AWS database services and scalable infrastructure. Through hands-on practice with Amazon RDS, Amazon DynamoDB, Amazon Lightsail, Amazon EC2 Auto Scaling, and Elastic Load Balancer, I learned how to build stable, scalable, and highly available applications. These skills provide an important foundation for modern architectures, container technologies, and DevOps services in the following weeks.
  • Successfully practiced Amazon DynamoDB, designed database tables, selected appropriate Partition Keys, and performed complete CRUD operations. Understood the advantages of DynamoDB for highly scalable, high-performance, and low-latency systems.
  • Became familiar with Amazon Lightsail, deployed a WordPress website, configured its environment, and managed the application through the Lightsail console. Understood how Elastic Load Balancer (ELB) distributes incoming traffic across multiple servers to improve availability and fault tolerance.
  • Understood how Amazon EC2 Auto Scaling works, created an Auto Scaling Group and Launch Template, and configured scaling policies based on resource utilization. Learned the principles of High Availability on AWS and the role of combining Auto Scaling with Load Balancing to build scalable and resilient cloud systems.

Week 2 Evaluation:

  • The second week focused on expanding my AWS resource management skills through AWS CLI while gaining deeper knowledge of AWS database services and scalable infrastructure. Through practical exercises with Amazon RDS, Amazon DynamoDB, Amazon Lightsail, Amazon EC2 Auto Scaling, and Elastic Load Balancer, I developed a better understanding of how to build stable, scalable, high-performance, and highly available applications. The knowledge acquired this week provides a solid foundation for implementing modern architectures, containerized applications, and DevOps services in the following weeks.
