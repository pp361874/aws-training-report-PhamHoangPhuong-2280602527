Week 7 Worklog

Week 7 Objectives:

  • Operate and optimize a real-world CI/CD pipeline by automating the complete application delivery process from GitHub to Amazon ECS.
  • Improve database availability and scalability by configuring disaster recovery mechanisms for Amazon RDS and optimizing performance and cost for Amazon DynamoDB.
  • Explore Big Data Analytics by learning how to collect, process, and analyze real-time streaming data using AWS services.

Week 7 Achievements:

  • Successfully built a CI/CD pipeline on AWS by integrating GitHub with AWS CodePipeline, AWS CodeBuild, and AWS CodeDeploy to automate application build and deployment processes. Learned how to validate the CI/CD pipeline by updating source code, monitoring build and deployment stages, and verifying automatic application updates on Amazon ECS.
  • Learned how to use Global Secondary Indexes (GSI) to optimize query performance in Amazon DynamoDB, and understood how index design affects database performance and operational costs. Explored Amazon Kinesis and understood the process of collecting, streaming, and processing real-time data, as well as its applications in Big Data systems and infrastructure monitoring.

Week 7 Evaluation:

  • The seventh week focused on completing a CI/CD pipeline, improving database availability, and exploring AWS data processing services. Through building an automated deployment pipeline, configuring Amazon RDS Multi-AZ, optimizing Amazon DynamoDB with Global Secondary Indexes, and studying Amazon Kinesis, I gained a better understanding of how to design modern cloud systems that support automated deployment, high availability, and large-scale real-time data processing.
